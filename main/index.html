Static Timing Analysis (STA)

    Overview of Static Timing Analysis

    Static Timing Analysis (STA) checks the worst-case propagation delay for all possible input vectors in digital circuits, ensuring timing constraints are met without needing actual input data.

    Advantages:

    • Significantly faster than timing-driven gate-level simulation.
    • Exhaustive testing of all timing paths under constraints.

    Disadvantages:

    • Does not verify circuit functionality.
    • Requires accurate definition of timing constraints.

    Timing Paths

    A timing path in STA represents the route from a startpoint to an endpoint through clock pins, primary inputs (PI), primary outputs (PO), and other elements.

    • Startpoint: Origin of the timing path, typically a clock pin.
    • Endpoints: Destination of the timing path, often a primary output or clock pin.
    • Input Ports (PI): External inputs affecting timing.
    • Output Ports (PO): External outputs affected by timing.
    • Memories / Hard Macros: Specific components with predefined timing characteristics.

    Goals of Static Timing Analysis

    Key objectives of STA include:

    • Verification of maximum and minimum delay constraints across all paths.
    • Starting with a gate-level netlist and applying timing models from the library.
    • Reporting violations of timing constraints and their locations for corrective action.
    • Comprehensive analysis to identify and prioritize timing issues.

    Static Timing Analysis ensures that digital circuits meet timing requirements critical for proper operation.
